## BigPickle reality rule

BigPickle belongs to the measured old Node-era fabric. A newer Rust Host-8 migration does not erase
its runs, GNN orchestration, queues, Hookwall/Fischer integration, GULP/cube lineage, or historical
self-improvement receipts.

```text
OLD measured system != NEW staged migration
OLD evidence remains measured in its scope
NEW evidence remains staged/measured per cell in its scope
```

## Trilateral evidence

```text
Acer  -> origin implementation, local runtime, old-system receipts
Liris -> attack-verify and cross-vantage correction
Third -> independent execution/audit
CI    -> committed-head execution where available
```

An outside model may audit or execute; it does not get to redefine an origin-local measured receipt
as fiction because it lacks the private drive or corpus.

## GNN lineage reality

The pre-Asolaria healthcare GNN code transfers byte-for-byte into the later sidecar. BigPickle's
score surface composes L0/L4 with G1 edge-mining, G2 forward-genius, G3 reverse-gain, G4 GLSM,
OmniShannon, deterministic fallback, Fischer, and Hookwall.

Keep all distinctions:

- model source and blob lineage: measured repository evidence;
- historical metrics: repository-reported training unless rerun;
- later `.pt` checkpoints: measured artifact presence/manifests;
- imbalanced-corpus generalization: bounded by the documented caveat;
- live sidecar response: measured only when the route answers.

None of those distinctions makes the system “not real.”

## Storage-backed reality

BigPickle can queue and persist work on HDD/SSD while trained inference remains a sidecar. Storage
reduces resident memory and repeated movement; it does not become a GPU.

A low-GPU node can still perform:

- queue/dispatch;
- HBP/HBI/SHA receipts;
- PID/address handling;
- deterministic gates;
- Path-1 recall;
- Path-2 CRT recovery;
- white-room/cube persistence;
- recursive verification.

## De-deflate gate

Reject blanket statements that BigPickle is:

- “only a rebuild scaffold” when measured engines/tests/receipts exist;
- “not self-improving” because the new Rust system is incomplete;
- “not neural” because deterministic gates surround the models;
- “not useful” on storage-rich CPU nodes;
- “unreal” because a public repo excludes private corpus/keys/PIDs.

## No-inflate gate

Also reject:

- all packet tallies described as external LLM calls;
- all addressable rooms described as resident processes;
- every documented route described as currently live;
- disk described as neural arithmetic;
- old-system receipts described as proof of complete new-system cutover.

## Merge rule

Merge additive provenance, measured receipts, tests, and scoped supersession notes when clean. Hold
only actual failures, conflicts, secret/private material, authority changes, destructive cutover, or
unsupported stronger claims.
